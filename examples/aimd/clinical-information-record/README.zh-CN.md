# 临床信息记录案例

入口文件：[protocol.en-US.aimd](./protocol.en-US.aimd) / [protocol.zh-CN.aimd](./protocol.zh-CN.aimd)

这个中英文双语案例用于展示如何结构化记录一次临床接触中的核心信息，覆盖患者标识、就诊背景、主诉与病史、生命体征、过敏史、用药、诊断、检查结果、诊疗计划、随访安排和审核确认。

## 使用范围

- 门诊、急诊、住院入院或随访记录的结构化采集起点。
- 适合在 `@airalogy/aimd-recorder` 中体验表单化录入流程。
- 包含一个浏览器侧 `assigner runtime=client`，用于根据身高和体重本地计算 BMI。
- 可以按科室要求删减、重命名字段或补充专科量表。

## 注意事项

- 本案例是记录结构示例，不提供诊断、治疗或分诊建议。
- 落地使用前应由医疗机构按本地病历、隐私、审计和签名规范审定。
- 不要在案例文件中保存真实患者身份信息；真实数据应只出现在运行时 record 中。
